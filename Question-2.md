# 🚀 Q2: Add Sidecar Container to Existing Deployment

---

## 📘 Question

Update the existing deployment `wordpress`, adding a sidecar container named `sidecar` using the image `busybox:stable`.

---

## 📋 Requirements

- Sidecar should run:

```/bin/bash -c "tail -f /var/log/wordpress.log"
```
Use a shared volume mounted at /var/log
Ensure log file wordpress.log is accessible to both containers
✅ Answer
🔹 Solution 1: Basic Sidecar Concept (Pod YAML)

This demonstrates how a sidecar container works using a standalone Pod.

```
apiVersion: v1
kind: Pod
metadata:
  name: wordpress
  labels:
    run: wordpress

spec:
  volumes:
  - name: log
    emptyDir: {}

  containers:

  - name: wordpress
    image: wordpress
    volumeMounts:
    - name: log
      mountPath: /var/log

  - name: sidecar
    image: busybox:stable
    command: ["/bin/sh", "-c", "tail -f /var/log/wordpress.log"]
    volumeMounts:
    - name: log
      mountPath: /var/log
```
# 🔹 Solution 2: Update Existing Deployment (Recommended)

## ⚙️ Step 1: Create Deployment

```kubectl create deployment wordpress --image=wordpress```

## 🔍 Step 2: Verify Pod
```kubectl get pods```

## ✏️ Step 3: Edit Deployment
```kubectl edit deployment wordpress```

## 🧩 Step 4: Add Sidecar + Volume

Update the deployment with:

```
# Final Deployment File with Sidecar Container

apiVersion: apps/v1
kind: Deployment

metadata:
  name: wordpress
  namespace: default
  labels:
    app: wordpress
  annotations:
    deployment.kubernetes.io/revision: "2"

spec:
  replicas: 1

  # Selector to match pods
  selector:
    matchLabels:
      app: wordpress

  # Deployment update strategy
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%

  template:
    metadata:
      labels:
        app: wordpress

    spec:
      containers:

      # 🔹 Sidecar Container (NEW)
      - name: sidecar
        image: busybox:stable
        command:
        - /bin/sh
        - -c
        - tail -f /var/log/wordpress.log
        volumeMounts:
        - name: log
          mountPath: /var/log

      # 🔹 Main WordPress Container
      - name: wordpress
        image: wordpress
        imagePullPolicy: Always
        resources: {}
        terminationMessagePath: /dev/termination-log
        terminationMessagePolicy: File

        # Mount same volume to share logs with sidecar
        volumeMounts:
        - name: log
          mountPath: /var/log

      # 🔹 Shared Volume between containers
      volumes:
      - name: log
        emptyDir: {}

      # Pod-level configurations
      dnsPolicy: ClusterFirst
      restartPolicy: Always
      schedulerName: default-scheduler
      securityContext: {}
      terminationGracePeriodSeconds: 30
```
